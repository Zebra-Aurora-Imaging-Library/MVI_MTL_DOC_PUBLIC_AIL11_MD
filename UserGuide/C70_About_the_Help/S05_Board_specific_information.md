# System-specific information and function availability

Some parameter settings and information in the [Aurora Imaging Library Reference](../../Reference/index.md) are only applicable when using certain Aurora Imaging Library systems. This information is referred to as **System-specific information**. An Aurora Imaging Library system typically refers to a set of hardware components capable of grabbing, storing, processing, and/or displaying images (for example, a Zebra Rapixo system consists of the imaging board, the CPU, and a display board). Aurora Imaging Library systems are allocated using [`MsysAlloc`](../../Reference/sys/MsysAlloc.md).

Click the *[Image: Gear_Button_New_Help.png]*

 button to open the **Help view settings** dialog, which allows you to customize the display of system-specific information.

> **Note:** The Aurora Imaging Library release notes, accessible from Aurora Imaging Control Center and/or the Zebra Aurora Imaging Library product support webpages (for example, [zebra.com/ail-info](https://zebra.com/ail-info)), can provide additional documentation, such as new or modified features, systems, and products.

## Availability

Aurora Imaging Library and Aurora Imaging Library Lite share core functionality, including application and system control, image acquisition and archiving, graphical annotations, and basic image processing operations that are typically useful to preprocess grabbed images. In the [Reference](../../Reference/index.md), each function has an availability note in the menu bar.

When you set your product to Aurora Imaging Library Lite in the **Help view settings** dialog, supported functions are listed in blue on the module's information page. To access this page, click the module's prefix under the Reference heading in the **Contents** tab (for example, click "im" to see the list of functions in the Image Processing module).
