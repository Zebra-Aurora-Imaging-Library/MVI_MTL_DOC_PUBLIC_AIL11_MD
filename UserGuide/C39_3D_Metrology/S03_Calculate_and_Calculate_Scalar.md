# Calculating geometric and scalar measurements

The Aurora Imaging Library 3D Metrology module can perform geometric and scalar measurements for 3D geometry objects, using [`M3dmetFeatureEx`](../../Reference/3dmet/M3dmetFeatureEx.md) and [`M3dmetFeature`](../../Reference/3dmet/M3dmetFeature.md). For example, you can use either of these functions to find the closest distance between a point and another 3D geometry. Note that [`M3dmetFeatureEx`](../../Reference/3dmet/M3dmetFeatureEx.md) can calculate both the distance and the point on the second geometry. If you only need the distance (and not the associated point), use [`M3dmetFeature`](../../Reference/3dmet/M3dmetFeature.md), which returns the distance measurement directly. In general, if you require a calculated 3D geometry, use [`M3dmetFeatureEx`](../../Reference/3dmet/M3dmetFeatureEx.md); if you require a scalar measurement only, use [`M3dmetFeature`](../../Reference/3dmet/M3dmetFeature.md).

Note that passing a result buffer to [`M3dmetFeatureEx`](../../Reference/3dmet/M3dmetFeatureEx.md) ensures that you can retrieve both scalar measurements and any computed geometry. If instead you pass a 3D geometry object to hold the result, you will only obtain a defined 3D geometry; no scalar measurement is saved.

Besides 3D geometries, you can also perform some operations on transformation matrix objects. For example, you can interpolate between matrices, calculate the distance between origins defined in the matrices, or calculate the angle between their rotations.

## Steps to performing a M3dmetFeatureEx or M3dmetFeature operation

The following steps provide a basic methodology for performing a [`M3dmetFeatureEx`](../../Reference/3dmet/M3dmetFeatureEx.md) or [`M3dmetFeature`](../../Reference/3dmet/M3dmetFeature.md) operation:

1. Obtain the source geometries or matrices. You will likely already have at least one source 3D geometry or matrix with which you want to work. To create a new 3D geometry or matrix, see [Using the 3D Geometry module](../C44_Using_the_3D_Geometry_module/ChapterInformation.md).
2. For an [`M3dmetFeatureEx`](../../Reference/3dmet/M3dmetFeatureEx.md) operation, allocate a result object:
   Note that, for operations that result in a transformation matrix (for example, when interpolating between two matrices with [`M_INTERPOLATION`](../../Reference/3dmet/M3dmetFeatureEx.md)), you must allocate a result buffer to store the result.
   - To allocate a 3D metrology result buffer, use [`M3dmetAllocResult`](../../Reference/3dmet/M3dmetAllocResult.md) with [`M_CALCULATE_RESULT`](../../Reference/3dmet/M3dmetAllocResult.md).
   - To allocate a 3D geometry object, use [`M3dgeoAlloc`](../../Reference/3dgeo/M3dgeoAlloc.md) with [`M_GEOMETRY`](../../Reference/3dgeo/M3dgeoAlloc.md).
   [`M3dmetFeature`](../../Reference/3dmet/M3dmetFeature.md) returns a single value, so it does not take a result object.
3. Call [`M3dmetFeatureEx`](../../Reference/3dmet/M3dmetFeatureEx.md) or [`M3dmetFeature`](../../Reference/3dmet/M3dmetFeature.md) with the source 3D geometries or matrices, the operation to perform, and the object/location in which to store results.
4. If necessary, retrieve results from the 3D metrology result buffer using [`M3dmetGetResult`](../../Reference/3dmet/M3dmetGetResult.md).
5. If required, copy results from your 3D metrology result buffer into a 3D geometry object or a transformation matrix object, using [`M3dmetCopyResult`](../../Reference/3dmet/M3dmetCopyResult.md). For example, to retrieve a resulting 3D geometry after a [`M3dmetFeatureEx`](../../Reference/3dmet/M3dmetFeatureEx.md) operation, you can pass the result buffer and a 3D geometry object to [`M3dmetCopyResult`](../../Reference/3dmet/M3dmetCopyResult.md) to obtain a defined 3D geometry.

In the following subsections, the possible [`M3dmetFeatureEx`](../../Reference/3dmet/M3dmetFeatureEx.md) and [`M3dmetFeature`](../../Reference/3dmet/M3dmetFeature.md) operations are presented in tables, arranged into groups of operations that only compute a 3D geometry or matrix, operations that only compute a scalar or other non-geometric value, and operations that compute a 3D geometry, a non-geometric value (typically, a scalar value), or both.

## Operations that compute a 3D geometry, a related scalar value, or both.

Some operations compute a 3D geometry, a related non-geometric value (typically, a scalar value), or both. For example, [`M_SHORTEST_LINE`](../../Reference/3dmet/M3dmetFeatureEx.md) calculates the shortest line between two source lines; you can choose to obtain the computed 3D line geometry, its length, or both. These operations are available when using [`M3dmetFeatureEx`](../../Reference/3dmet/M3dmetFeatureEx.md) (geometry and scalar results) or [`M3dmetFeature`](../../Reference/3dmet/M3dmetFeature.md) (scalar result only). Typical scalar results include the number of resulting 3D geometries, as well as certain distance measurements.

> **Note:** Note that, for some of the operations listed below, you can also retrieve information about the computation, such as the degree to which the source geometries are parallel or perpendicular, if applicable. You can use these results to determine edge cases. For example, when calculating the intersection of two planes, the resulting line might be too far away to be useful, which can occur when the source planes are almost parallel.

When using [`M3dmetFeatureEx`](../../Reference/3dmet/M3dmetFeatureEx.md), if you want to obtain both 3D geometry and scalar results, you must specify a result buffer; if you specify a 3D geometry result object when using this function, you will obtain a 3D geometry only.
