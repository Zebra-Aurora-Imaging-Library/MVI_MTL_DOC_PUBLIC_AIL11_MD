# Guidelines for managing an images dataset

This section relates some guidelines, recommendations, and details to help you make the most efficient use of an images dataset. The number of images in a dataset, and the various purposes it can serve, make it likely that you will want to make your dataset portable and well organized. This will allow you to easily reuse your dataset, or combine it with other datasets.

## Data on disk

An images dataset stores the bulk of its data (images) on disk. For example, you must save your entry images on disk before adding them as entries in the dataset (an images dataset holds the paths to each entry image, not the image itself).

> **Note:** It is recommended to decouple dataset creation from usage. Since constructing a dataset involves disk storage, it should be done first, properly, and on its own. Once the dataset is ready, you can use it (by restoring or importing it) for training or prediction.

Data stored on disk should be well organized and follow a recommended folder structure which, among other guidelines, separates information according to the dataset to which it belongs. Segmentation, as well as operations such as data preparation, training, and prediction, automatically adds information related to datasets to disk (including images) according to the folders, paths, and structures specified below.

> **Note:** Although certain operations save new dataset information on disk, they do not typically delete it (with the exception of [training with a single dataset](S07_Guidelines_for_managing_an_images_dataset.md)). You must therefore be mindful of the resources required to save this data, and manage your disk storage accordingly.

## Folders

The variety of information that an images dataset can hold is organized on disk according to folders.

By default, some operations produce folders in the public user's document folder. To change this default location, use the **Aurora Imaging Configurator** utility. For example, this folder path stores the images, segmentation and object detection regions (masks), and scores produced by [`MclassPrepareData`](../../Reference/class/MclassPrepareData.md) and [`MclassTrain`](../../Reference/class/MclassTrain.md).

You can also specify a root path by calling [`MclassControl`](../../Reference/class/MclassControl.md) with [`M_ROOT_PATH`](../../Reference/class/MclassControl.md). This can be set while the dataset is empty, or if the dataset file paths are made relative ([`M_MAKE_FILE_PATHS_RELATIVE`](../../Reference/class/MclassControl.md)). This represents the top-most path for all other paths explicitly specified in an image dataset (for example, entry images). All images datasets require, at least, entry images.

An images dataset that is for segmentation holds additional data, such as the segmentation masks containing the ground truth information, which is saved at the folder specified by the [`M_REGION_MASKS_FOLDER`](../../Reference/class/MclassControl.md) control. The path of this folder must be set before you call [`MclassEntryAddRegion`](../../Reference/class/MclassEntryAddRegion.md) with regions based on images. Segmentation prediction scores that [`MclassPredict`](../../Reference/class/MclassPredict.md) produces (every pixel has a class score) are saved with the destination dataset at the folder specified by the [`M_SEGMENTATION_FOLDER`](../../Reference/class/MclassControl.md) control. You must also set the path of this folder.

Datasets for other machine learning tasks, such as object detection, can also hold additional/similar information. Note, prediction scores for object detection (only instances are scored) are saved in the dataset.

### Data preparation and training with a single dataset

New images and masks generated by [`MclassPrepareData`](../../Reference/class/MclassPrepareData.md) are saved at the folder specified by the [`M_PREPARED_DATA_FOLDER`](../../Reference/class/MclassControl.md) control before their paths are added to the destination dataset.

Images and masks prepared when calling [`MclassTrain`](../../Reference/class/MclassTrain.md) with a single dataset are saved at the location specified by the [`M_TRAIN_DESTINATION_FOLDER`](../../Reference/class/MclassControl.md) control. The content of this folder is overwritten at each new training; prepared datasets must be copied to new datasets using [`MclassCopyResult`](../../Reference/class/MclassCopyResult.md)([`M_PREPARED_DEV_DATASET`](../../Reference/class/MclassCopyResult.md)/[`M_PREPARED_TRAIN_DATASET`](../../Reference/class/MclassCopyResult.md)). You should then ensure such data is made portable, as described in [Creating portable and properly structured datasets](S07_Guidelines_for_managing_an_images_dataset.md).

### Exporting and importing

The disk content of an images dataset that you exported using [`MclassExport`](../../Reference/class/MclassExport.md) with [`M_IMAGE_DATASET_FOLDER`](../../Reference/class/MclassExport.md) and [`M_COMPLETE`](../../Reference/class/MclassExport.md) is copied under the destination folder specified with the [`FileNameOrFolderPath`](../../Reference/class/MclassExport.md) parameter (the dataset from which you exported is left unchanged). Since this is a complete copy, all data, along with CSV files holding complementary information (such as authors and segmentation prediction results), is copied in a structured set of folders (all under the destination folder).

If you are importing data to an images dataset, the folder from which you are importing must adhere to the same type of folder structure the Aurora Imaging Library uses when exporting datasets. For more information about this folder structure and the content therein, see [Importing data from a folder or CSV file](S06_Importing_data_from_a_CSV_file_to_a_dataset_context.md).

### Consolidation

You can copy and reorganize a dataset according to a specified destination directory using the [`M_CONSOLIDATE_ENTRIES_INTO_FOLDER`](../../Reference/class/MclassControl.md) control. The structure of the directory is the same as when exporting a dataset, except it does not include class definition icons and CSV files. The following is an example:

|   |   |
| --- | --- |
| ```<br/>DstFolder<br/>├───Icons¹<br/>│ ├───Class1.mim<br/>│ ├───Class2.mim<br/>├───Images <br/>│ ├───Class1¹<br/>│ │ ├───Image1.mim<br/>│ │ └───Image2.mim<br/>│ ├───Class2<br/>│ │ ├───Image1.mim<br/>│ │ └───Image2.mim<br/>│ ├───Image1.mim²<br/>│ └───Image2.mim<br/>├───Masks³<br/>│ ├───GUID1.mim<br/>│ └───GUID2.mim<br/>├───Segmentations⁴<br/>│ ├───GUID1_0.mbufi<br/>│ ├───GUID2_0.mbufi<br/>│ ├───GUID3_0.mbufi<br/>``` | ¹Sub-folders at this level contain images labeled, with respect to image classification, for each class. Anomaly detection follows the same convention as image classification, with only 2 possible classes. That is, the vast majority of your images should be non-anomalous (one type of class), while some images used for testing can be anomalous (another type of class). ²Images labeled for segmentation and object detection are under the _Images_ folder. ³Contains the segmentation ground truth masks. ⁴Contains the segmentation prediction score buffers. |

In general, consolidation ([`M_CONSOLIDATE_ENTRIES_INTO_FOLDER`](../../Reference/class/MclassControl.md)) means that, within the destination folder, images of the dataset are copied to the images folder by class. If the dataset holds segmentation or object detection data, mask regions are copied in the _Masks_ folder and segmentation scores will be copied in the _Segmentations_ folder (object detection scores are in the dataset). The paths of all entries, masks, segmentation and object detection files, are updated as well as the _Masks_ ([`M_REGION_MASKS_FOLDER`](../../Reference/class/MclassControl.md)) and _Segmentations_ ([`M_SEGMENTATION_FOLDER`](../../Reference/class/MclassControl.md)) folders. Consolidation does not affect the source files.

> **Note:** The disk content of datasets consolidated using [`MclassControl`](../../Reference/class/MclassControl.md)([`M_CONSOLIDATE_ENTRIES_INTO_FOLDER`](../../Reference/class/MclassControl.md)) is copied under the destination folder and all the paths in the dataset are updated to reflect the changes.

## Creating portable and properly structured datasets

When creating an images dataset, it is recommended that it is portable (for example, easily transferred between computers) and properly structured. Such datasets are simpler to manage and expand which, in the long run, will save you time, since datasets are often fluid and changing (they are rarely static and unaltered).

In terms of portability and structure, it is highly recommended that you export and import your datasets using [`MclassExport`](../../Reference/class/MclassExport.md) and [`MclassImport`](../../Reference/class/MclassImport.md). Specifically, you can:

1. Write the dataset to an empty folder, using [`MclassExport`](../../Reference/class/MclassExport.md) with [`M_IMAGE_DATASET_FOLDER`](../../Reference/class/MclassExport.md) and [`M_COMPLETE`](../../Reference/class/MclassExport.md) (this exports the entire dataset, including all CSV files).
   You can now manage the dataset as a folder, such as compressing it or transferring it to another computer.
2. Read the exported dataset folder back into a dataset, using [`MclassImport`](../../Reference/class/MclassImport.md) with [`M_IMAGE_DATASET_FOLDER`](../../Reference/class/MclassImport.md) and [`M_COMPLETE`](../../Reference/class/MclassImport.md)(this imports the entire dataset, including all CSV files).

## Alternatives to import/export for portability (legacy)

If you do not want to follow the recommended procedure of exporting and importing your datasets ([`MclassExport`](../../Reference/class/MclassExport.md) / [`MclassImport`](../../Reference/class/MclassImport.md)), you can try to adhere to the following guidelines instead.

The initial steps for alternatively creating a portable and structured dataset are as follows, for image classification, anomaly detection, segmentation, and object detection:

| Steps | Notes |
| --- | --- |
| 1. Allocate a new dataset, using [`MclassAlloc`](../../Reference/class/MclassAlloc.md)([`M_DATASET_IMAGES`](../../Reference/class/MclassAlloc.md)).<br/>2. Save the dataset to an empty folder, using [`MclassSave`](../../Reference/class/MclassSave.md).<br/>3. Reload the dataset saved on disk, using [`MclassRestore`](../../Reference/class/MclassRestore.md), or [`MclassStream`](../../Reference/class/MclassStream.md) with [`M_LOAD`](../../Reference/class/MclassStream.md), to properly set the special path_///DATASET///_.<br/>4. Make the dataset file paths relative to _///DATASET///_, using [`MclassControl`](../../Reference/class/MclassControl.md) and [`M_ROOT_PATH`](../../Reference/class/MclassControl.md) set to _AIL_TEXT("///DATASET///"))_.<br/>5. In the dataset folder, create the subfolder _Images_.¹ | ¹Once you create the _Images_ subfolder, you should follow the steps specific to the task you are forming ([Image classification](S07_Guidelines_for_managing_an_images_dataset.md), [Anomaly detection](S07_Guidelines_for_managing_an_images_dataset.md), [Segmentation](S07_Guidelines_for_managing_an_images_dataset.md), or [Object detection](S07_Guidelines_for_managing_an_images_dataset.md)). |

Regardless of whether an images dataset is for image classification, anomaly detection, segmentation, or object detection, note that:

- When the dataset comes from a file on a disk, the special path _///DATASET///_ refers to the parent directory of the dataset file (.mclassd). For example, if your dataset is saved in the location _C:/Documents/Dataset.mclassd_, _///DATASET///_ will be interpreted as _C:/Documents_. The file name itself is not included in _///DATASET///_, which helps make the dataset portable.

### Special DATASET path alternative

The following steps provide an alternative for making datasets portable. Although more complex, this does not involve CSV files or importing.

1. Save the dataset to an empty folder, using [`MclassSave`](../../Reference/class/MclassSave.md).
2. Reload the dataset saved on disk using [`MclassRestore`](../../Reference/class/MclassRestore.md), or [`MclassStream`](../../Reference/class/MclassStream.md) with [`M_LOAD`](../../Reference/class/MclassStream.md).
3. Consolidate the dataset into that folder, using [`MclassControl`](../../Reference/class/MclassControl.md) with [`M_CONSOLIDATE_ENTRIES_INTO_FOLDER`](../../Reference/class/MclassControl.md) and specify AIL_TEXT("///DATASET///")).
4. Make the dataset file paths relative to _///DATASET///_, using [`MclassControl`](../../Reference/class/MclassControl.md) with [`M_MAKE_FILE_PATHS_RELATIVE`](../../Reference/class/MclassControl.md).
5. Call [`MclassSave`](../../Reference/class/MclassSave.md) to update the dataset saved on disk with these new changes.
   You can now manage the dataset as a folder, such as compressing it or transferring it to another computer.
6. Read the dataset back into Aurora Imaging Library, using [`MclassRestore`](../../Reference/class/MclassRestore.md).
