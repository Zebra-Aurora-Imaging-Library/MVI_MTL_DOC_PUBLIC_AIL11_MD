# Retrieving the orientation and center point or position of a 3D object

You can retrieve the orientation and center point or position of a point cloud, depth map, or 3D geometry object.

If you are unsure of where to start or which module to use, consult the following for suggestions.

### Dealing with a very large or complicated point cloud

In the case of a very large and complex scene, start with the [3D Blob Analysis](../C36_3D_Blob_analysis/ChapterInformation.md) module to remove extraneous information and isolate areas of interest. Then, use the [3D Model Finder](../C37_3D_Model_finder/ChapterInformation.md), [3D Metrology](../C39_3D_Metrology/ChapterInformation.md), or [3D Registration](../C40_3D_Registration/ChapterInformation.md) module. If the scene is primarily composed of boxes, an initial pass with 3D blob analysis might not be needed and you can use 3D model finder directly. Note that the 3D Model Finder module is slow when the complexity of the scene is high, but is similar to the 3D Blob Analysis module when searching for boxes. If the objects are boxes, you can use the 3D Model Finder module to avoid additional steps without increasing the search time.

Once the larger point cloud is segmented into blobs, use the [3D Model Finder](../C37_3D_Model_finder/ChapterInformation.md) module to find the pose of all blobs or use the [3D Metrology](../C39_3D_Metrology/ChapterInformation.md) or [3D Registration](../C40_3D_Registration/ChapterInformation.md) module to find the pose of specific blobs. For example, if the object in a blob is a simple geometry (like a sphere), use 3D metrology. If 3D metrology is not suitable, but you are confident that an isolated blob represents your target, use 3D registration.

### Remarks

- Typically, a 3D module's main computation function (for example, [`M3dmodFind`](../../Reference/3dmod/M3dmodFind.md), [`M3dmetFit`](../../Reference/3dmet/M3dmetFit.md)) performs the initial object detection (such as finding a model occurrence). Once calculated, you can retrieve relevant results such as [`M_CENTER_...`](../../Reference/3dmod/M3dmodGetResult.md). For a module-specific listing of ways to retrieve a 3D object's orientation and center point or position, see [Module-specific values for retrieving an object's pose](S03_Retrieving_the_orientation_and_center_point.md).
- Your target and/or reference point clouds might require sampling to bring their respective resolutions to the same density. See [](S02_Implementation_considerations.md).
- Using 2D finder modules on your data can save computation time, since some 3D operations are costly in this respect. For example, it could be more efficient to use the 2D [Model Finder](../C08_Geometric_Model_Finder/ChapterInformation.md) module to find an object before deploying the [3D Registration](../C40_3D_Registration/ChapterInformation.md) module to establish the object's precise pose in 3 dimensions.
- To make 3D data available in 2D, you can project a point cloud into a depth map using [`M3dimProject`](../../Reference/3dim/M3dimProject.md) or [`M3dimProjectEx`](../../Reference/3dim/M3dimProjectEx.md). To project points onto a fitted plane, you might require a fixturing matrix that provides the transformation required to fixture the points such that points are projected to the plane. See [](../C45_Fixturing_in_3D/S01_Fixturing_in_3D_overview.md) for more information.
- When using the 3D Model Finder module, if there is a lot of occlusion you should set the maximum coverage ([`M3dmodControl`](../../Reference/3dmod/M3dmodControl.md) with [`M_COVERAGE_MAX`](../../Reference/3dmod/M3dmodControl.md)). Otherwise, the default settings are typically sufficient.

For a discussion of other considerations when working with 3D data, see [Implementation considerations](S02_Implementation_considerations.md).

## Module-specific values for retrieving an object's pose

Typically, a 3D module's [`M...GetResult`](../../Reference/3dmet/M3dmetGetResult.md) function can return the orientation and center point of a point cloud, depth map, or 3D geometry object.

In general for a 3D geometry, once you have found the required geometry, fitted one into your scene, or copied results into a 3D geometry object (for example, from a 3D blob analysis calculation), its pose is available. Note that some geometries are available only once you copy them out of a result buffer. For example, when searching for a box model using [`M3dmodFind`](../../Reference/3dmod/M3dmodFind.md), a successful occurrence is stored in a result buffer. To access the box, you must copy it into a 3D geometry object (in this case, using [`M3dmodCopyResult`](../../Reference/3dmod/M3dmodCopyResult.md)) before you can retrieve its orientation.

The following table lists the ways to retrieve a 3D object's orientation and center point or position, depending on the module used.

| Module | Orientation | Center point or position |
| --- | --- | --- |
|  |
| 3dmod - For a found occurrence, obtained using [`M3dmodFind`](../../Reference/3dmod/M3dmodFind.md). | Use [`M3dmodGetResult`](../../Reference/3dmod/M3dmodGetResult.md) with: - [`M_AXIS_...`](../../Reference/3dmod/M3dmodGetResult.md) to get a cylinder's orientation.<br/>- [`M_NORMAL_...`](../../Reference/3dmod/M3dmodGetResult.md) to get a plane's orientation. For a box model occurrence, you can use [`M3dmodCopyResult`](../../Reference/3dmod/M3dmodCopyResult.md) with [`M_GEOMETRY`](../../Reference/3dmod/M3dmodCopyResult.md) to establish a box with the same orientation. Then, use [`M3dgeoCopy`](../../Reference/3dgeo/M3dgeoCopy.md) with [`M_BOX_ORIENTATION`](../../Reference/3dgeo/M3dgeoCopy.md) to copy, into a transformation matrix object, the transformation values that determine the box's orientation. To find the tilt angle of a box with respect to the XY-plane, use [`M3dgeoConstruct`](../../Reference/3dgeo/M3dgeoConstruct.md) with [`M_PLANE`](../../Reference/3dgeo/M3dgeoConstruct.md) and [`M_FACE`](../../Reference/3dgeo/M3dgeoConstruct.md). You can specify the face that is located furthest from the XY-plane (maximum Z) to construct a plane at that face. Then, call [`M3dmetFeature`](../../Reference/3dmet/M3dmetFeature.md) with [`M_ANGLE`](../../Reference/3dmet/M3dmetFeature.md) to calculate the tilt angle. | 3dreg - For point clouds for which a registration was computed using [`M3dregCalculate`](../../Reference/3dreg/M3dregCalculate.md). | Use [`M3dregCopyResult`](../../Reference/3dreg/M3dregCopyResult.md) with: - [`M_REGISTRATION_MATRIX`](../../Reference/3dreg/M3dregCopyResult.md) to find the transformation required to align the two point clouds. | — |
| 3dblob - For a blob, obtained using [`M3dblobSegment`](../../Reference/3dblob/M3dblobSegment.md), and for which the appropriate features have been calculated using [`M3dblobCalculate`](../../Reference/3dblob/M3dblobCalculate.md). | Use [`M3dblobGetResult`](../../Reference/3dblob/M3dblobGetResult.md) with: - [`M_AVERAGE_NORMAL_...`](../../Reference/3dblob/M3dblobGetResult.md) to get the average normal orientation of the blob's points<br/>- [`M_PRINCIPAL_AXIS_...`](../../Reference/3dblob/M3dblobGetResult.md) to get the components of the blob's three principal axes.<br/>- [`M_SEMI_ORIENTED_BOX_ANGLE`](../../Reference/3dblob/M3dblobGetResult.md) to get the angle of the blob's semi-oriented bounding box.
